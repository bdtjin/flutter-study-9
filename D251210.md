# 커밋 태그

- [컨벤셔널커밋] [https://www.conventionalcommits.org/ko/v1.0.0/]

## 커밋태그 종류

### 자주 사용되는 것들
- `feat`: 새로운 기능 추가했을 때
  - ex. `feat: 과제 1번 추가`
- `fix`: 버그 수정했을 때
  - ex. `fix: 문자열 출력 안 되는 현상 수정`
- `docs`: 코드의 수정없이 문서만 수정했을 때
  - ex. `docx: README 업데이트`
- `refactor`: 코드 리팩토링 했을 때
  - ex. `refactor: StudentScore 클래스 리팩토링`
-` WIP`: Working In Progress. 현재 작업이 완료되지 않았지만 커밋해야할 때 사용

### 추가
- `chore`: 귀찮은 작업, 예들 들어 패키지 버전 엡데이트 등
- `ci`: 지속적 통합(CI) 관련된 구성이 수정되었을 때
- `style`: UI 스타일 관련된 파일 수정되었을 때, 주로 프론트엔드에서 css 수정하고 많이 사용
